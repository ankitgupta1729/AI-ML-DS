The notion of "state" in LangGraph is crucial. To get an intuition of the LangGraph state, let's use an illustration:



In the example above, the conversation starts with a simple greeting from the user ("Hi"). This message is addressed to the Primary assistant, which we manually set as default entry node of the graph (we will implement this a few lectures later).

The Primary assistant then responds with "Hi, how can I assist you today?". This message goes directly to the user.

The user then responds, "I want information about my bills". Since the last assistant that responded to the user is the Primary assistant, this user's message is still addressed to the Primary assistant.

The Primary assistant then identifies that this message requires specialized handling by the Spending assistant, and redirects the flow towards it. The Spending assistant processes the user's message and replies "Are you interested in a specific billing period?", to narrow down the scope of the user's question.

The user then says "The past month". Now this is addressed to the Spending assistant directly and not the Primary assistant any more. That is because the dialog state changed.

Notice how as the conversation continues and the user receives responses from different assistants, the user inputs are addressed directly to these assistants dynamically.

To implement the above functionality we need to maintain a state object. The code is shown below:


from typing import Annotated, TypedDict, Literal, Optional
from langgraph.graph.message import AnyMessage, add_messages

def update_dialog_stack(left: list[str], right: Optional[str]) -> list[str]:
    """Push or pop the state."""
    if right is None:
        return left
    if right == "pop":
        return left[:-1]
    return left + [right]


class ChatbotState(TypedDict):
    messages: Annotated[list[AnyMessage], add_messages]
    user_info: str
    dialog_state: Annotated[
        list[Literal["primary", "spending", "recommendation"]],
        update_dialog_stack,
    ]


Our custom ChatbotState will be an object that dynamically updates the dialog_state field based on who was the last assistant that replied to the user. Also, it will maintain a messages field that will store all interactions between the user and the various assistants.
