Big-O notation answers one question: **as input size $n$ grows, how does the work grow?** GATE tests this relentlessly ‚ÄĒ in recurrences, loops, and data-structure operations. Get the intuition and you'll solve these in seconds.

## What Big-O really means

Big-O describes the **growth rate**, ignoring constants and lower-order terms. $3n^2 + 5n + 9$ is $O(n^2)$ because, for large $n$, the $n^2$ term dominates and constants don't matter.

The order you must know cold (slowest-growing to fastest):

$$O(1) < O(\log n) < O(n) < O(n\log n) < O(n^2) < O(2^n) < O(n!)$$

<Callout type="key">
$O(\log n)$ appears whenever you **halve** the problem each step (binary search, balanced BSTs). $O(n\log n)$ is the signature of good comparison sorts (merge sort, heap sort) ‚ÄĒ and the proven lower bound for comparison-based sorting.
</Callout>

## Reading loops

- One loop to $n$: $O(n)$.
- Nested loops, each to $n$: $O(n^2)$.
- A loop where the index **doubles** (`i *= 2`): $O(\log n)$.
- Halving the range each step: $O(\log n)$.

## Recurrences & the Master Theorem

Divide-and-conquer recurrences `T(n) = a¬∑T(n/b) + f(n)` are GATE staples. Compare `f(n)` with `n^(log_b a)`:

- If `f(n)` is **smaller** ‚áí `T(n) = őė(n^(log_b a))`.
- If they **match** ‚áí `T(n) = őė(n^(log_b a) ¬∑ log n)`.
- If `f(n)` is **bigger** (regularity holds) ‚áí `T(n) = őė(f(n))`.

Example: merge sort `T(n) = 2¬∑T(n/2) + O(n)` ‚Üí `n^(log‚āā2) = n` matches `f(n) = n` ‚Üí **őė(n log n)**. ‚úď

<Callout type="warning">
Don't confuse **O, őė, ő©**. O is an upper bound (‚Č§), ő© a lower bound (‚Č•), őė a tight bound (both). GATE deliberately mixes them in tricky options ‚ÄĒ read the symbol carefully.
</Callout>

## Common operation complexities (memorise the table)

- Array access $O(1)$, search $O(n)$.
- Hash table average $O(1)$, worst $O(n)$.
- Balanced BST insert/search/delete $O(\log n)$.
- Heap insert/extract $O(\log n)$, build-heap $O(n)$.
