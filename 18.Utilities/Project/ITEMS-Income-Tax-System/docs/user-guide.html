2. What is ITEMS?

ITEMS — the Income Tax Evaluation & Maintenance System — is application software that implements the core work of an income-tax practice. It replaces manual registers with a single, secure, computerised system that keeps client records, files original and revised income-tax returns, maintains firm accounts, and generates statutory reports.

Who is it for
Tax lawyers and practitioners who maintain returns for many clients across assessment years.
Why it helps
Removes paperwork and manual calculation, reduces errors, and makes years of records instantly searchable.

3. Objectives

  • Maintain the records of each client.
  • Based on the client's category, provide the forms to file the original return for a fiscal year.
  • Provide the facility to file a revised return in case of any mistake in the original return.
  • For firms, maintain the Trading A/c, Profit & Loss A/c and Balance Sheet.
  • Generate reports: return history of a client, returns filed in a fiscal, and revised returns by a client.

6. Client Information module

This is the master module. It maintains the basic information of each client — PAN, name, father's name, date of birth, address, category and ward/circle — and is responsible for adding, updating and deleting each client's record.

Add / edit a client

Click + Insert Client Record, fill the form, and Save. The PAN is validated (format ABCDE1234F) and used as the unique primary key. Deleting a client also removes all of that client's linked returns and firm accounts.

Figure 3 — Client list with instant search, category badges and row actions.

Figure 4 — Client entry form grouped into Identification, Contact and Classification.

7. Return Filing & tax computation

The Return module files the Original return for a client and assessment year, and lets you file a Revised return to correct mistakes. The system enforces the rule that a revised return can only be filed if an original return already exists for that year.

Filing a return

  • Choose the client (verified via PAN) and the return type.
  • Enter the assessment year and income under each head (salary, house property, business, capital gains, other sources).
  • The Computation of Tax panel updates live — total income, tax by slab, rebate u/s 87A, surcharge, 4% cess, and the final balance payable or refund.

Figure 5 — Return form with the live tax-computation panel on the right.

Tax figures use old-regime slabs (AY 2024-25) for individuals and a flat 30% for firms. They are illustrative for demonstration and training — verify against the current Finance Act before any real filing.

8. Firm accounts — Trading / P&L / Balance Sheet

When a client's category is Firm, three additional modules become relevant. Each has two sides that are totalled automatically, and the system shows whether they balance.

Trading Account
Debit (To) vs Credit (By) — opening stock, purchases, wages, gross profit vs sales, closing stock.
Profit & Loss A/c
All indirect expenses vs incomes, resolving to net profit or net loss.
Balance Sheet
Assets vs Liabilities & Capital, checked for balance.

Figure 6 — Balance Sheet records with automatic totals and a Balanced/Δ status.

9. Reports

The Report Generation module produces the statutory reports defined in the requirements. Generate a report, then use your browser's Print (Ctrl/Cmd+P) to save it as a PDF for the client file.

  • Return history of a client — every original and revised return, across years.
  • Returns filed in a fiscal — all clients for a chosen assessment year.
  • Revised returns by a client — total revised returns filed.

11. Security

  • Login — every module is behind a user id and password.
  • Password hashing — passwords are stored as SHA-256 hashes, never in clear text.
  • Audit log — the Oracle build records every insert, update and delete with user, table, key and timestamp.

12. Running the VB.NET + Oracle build

The production system is ASP.NET Web Forms (VB.NET) on Oracle. To run it:

  1. Run the SQL scripts in database/ in order (01 → 04) to create the schema, seed data, audit triggers and report views.
  2. Set the Oracle connection string in src/ITEMS/Web.config.
  3. Open src/ITEMS.sln in Visual Studio, restore the Oracle.ManagedDataAccess NuGet package, and run (F5) — or host on IIS / Oracle Cloud.

Full step-by-step instructions are in the project README.md.
